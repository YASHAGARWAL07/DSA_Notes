------------------------------------------------------------
1. WHAT IS GITHUB?
------------------------------------------------------------

GitHub is a cloud-based platform that hosts Git repositories.

Git = Version Control System  
GitHub = Hosting platform for Git repositories

It allows:
✔ Store code online
✔ Collaborate with others
✔ Track issues
✔ Review code
✔ CI/CD automation
✔ Open-source contribution


------------------------------------------------------------
2. GIT vs GITHUB
------------------------------------------------------------

Git:
- Software
- Installed locally
- Tracks changes

GitHub:
- Website/platform
- Stores repositories online
- Enables collaboration


------------------------------------------------------------
3. REPOSITORY (REPO)
------------------------------------------------------------

A repository is a project folder tracked by Git.

Types:
1) Public → Anyone can see
2) Private → Only invited users

Repo contains:
- Code files
- Commit history
- Branches
- README
- .gitignore
- License


------------------------------------------------------------
4. CREATING A REPOSITORY
------------------------------------------------------------

Steps:
1. Go to GitHub
2. Click "New Repository"
3. Enter name
4. Choose public/private
5. Add README (optional)

OR via terminal:

git remote add origin repo_url
git push -u origin main


------------------------------------------------------------
5. CONNECT LOCAL PROJECT TO GITHUB
------------------------------------------------------------

git init
git add .
git commit -m "Initial commit"

Add remote:
git remote add origin https://github.com/username/repo.git

Push:
git push -u origin main


------------------------------------------------------------
6. FORKING
------------------------------------------------------------

Fork = Copy someone else's repository into your account.

Used for:
- Open source contribution
- Making personal changes

Flow:
Fork → Clone → Make changes → Pull Request


------------------------------------------------------------
7. CLONE
------------------------------------------------------------

git clone repo_url

Downloads repository to local machine.


------------------------------------------------------------
8. BRANCHES IN GITHUB
------------------------------------------------------------

Default branch: main

Feature branches:
feature-login
bug-fix
dev

Best practice:
Never push directly to main in team projects.


------------------------------------------------------------
9. PULL REQUEST (PR)
------------------------------------------------------------

Pull Request = Request to merge your branch into another branch.

Used for:
✔ Code review
✔ Collaboration
✔ Discuss changes

Steps:
1. Push branch
2. Open PR on GitHub
3. Review
4. Merge


------------------------------------------------------------

3) Rebase and Merge

12. README FILE
------------------------------------------------------------
