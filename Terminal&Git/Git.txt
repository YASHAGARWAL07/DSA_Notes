------------------------------------------------------------
1. WHAT IS GIT?
------------------------------------------------------------

Git is a Distributed Version Control System (DVCS)
used to track changes in source code during development.

It allows:
✔ Track code history
✔ Collaborate with teams
✔ Manage branches
✔ Revert to previous versions
✔ Work offline


------------------------------------------------------------
2. VERSION CONTROL SYSTEM (VCS)
------------------------------------------------------------

Version Control System manages file changes over time.

Types:
1) Local VCS
2) Centralized VCS (e.g., SVN)
3) Distributed VCS (Git)

Git = Distributed
Every developer has full copy of repository.


------------------------------------------------------------
3. GIT ARCHITECTURE (VERY IMPORTANT)
------------------------------------------------------------

Git has 3 main areas:

1) Working Directory
   → Where you edit files.

2) Staging Area (Index)
   → Where files are prepared for commit.

3) Repository (.git folder)
   → Stores complete history.

Flow:

Working Directory → git add → Staging Area
Staging Area → git commit → Repository

------------------------------------------------------------

10. COMMITTING CHANGES
------------------------------------------------------------

Write meaningful commit messages


------------------------------------------------------------
11. VIEWING HISTORY
------------------------------------------------------------

→ Show commit history

→ Short version

------------------------------------------------------------
12. DIFFERENCE BETWEEN FILE VERSIONS
------------------------------------------------------------

git diff
→ Show unstaged changes

git diff --staged
→ Show staged changes


------------------------------------------------------------
13. BRANCHING (VERY IMPORTANT)
------------------------------------------------------------

What is Branch?
Independent line of development.

Default branch: main


Create branch:
git branch feature1

Switch branch:
git checkout feature1

OR (new method)
git switch feature1

Create + switch:
git checkout -b feature1


List branches:
git branch

Delete branch:
git branch -d feature1


------------------------------------------------------------
14. MERGING
------------------------------------------------------------

Merge branch into main:

git checkout main
git merge feature1

Merge conflict occurs when:
Same line modified in both branches.


------------------------------------------------------------
15. REMOTE REPOSITORY
------------------------------------------------------------

View remotes:
git remote -v

Add remote:
git remote add origin repository_url

Push code:
git push origin main

Pull code:
git pull origin main

Fetch changes:
git fetch


Difference:
fetch → downloads changes
pull → fetch + merge


------------------------------------------------------------
16. PUSHING FIRST TIME
------------------------------------------------------------

git push -u origin main

-u sets upstream branch


------------------------------------------------------------
17. STASHING
------------------------------------------------------------

Temporarily save changes without commit.

git stash
git stash list
git stash apply


------------------------------------------------------------
18. RESET & REVERT (INTERVIEW FAVORITE)
------------------------------------------------------------

git reset --soft HEAD~1
→ Remove last commit (keep changes)

git reset --hard HEAD~1
→ Remove commit & changes

git revert commit_id
→ Undo commit safely


Difference:
reset rewrites history
revert creates new commit


------------------------------------------------------------
19. REMOVE FILE FROM GIT
------------------------------------------------------------

git rm file.txt
→ Remove from repo

git rm --cached file.txt
→ Remove from Git but keep locally
